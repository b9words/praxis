#### **Gap 2: The Reading Experience is Passive**
A long markdown file feels like a textbook. We need to make the learning process itself more active and engaging.

**Solution: Enhance the Lesson Page (`app/(app)/library/curriculum/[domain]/[module]/[lesson]/page.tsx`)**

1.  **Interactive Blocks within Markdown:**
    *   **Action:** Enhance your `MarkdownRenderer.tsx` to recognize custom syntax for interactive components.
    *   **Example:**
        *   `>[!QUESTION]` block could render a non-graded multiple-choice quiz to check understanding.
        *   `>[!REFLECT]` block could render a private textarea for the user to type reflections, which could be saved with their progress.
        *   `>[!KEY-TAKEAWAY]` block could render a visually distinct callout box.

2.  **Seamless "Learn-Practice Loop":**
    *   **Action (Content):** At the end of each core lesson `.md` file, add a section that explicitly introduces a relevant case study.
    *   **Action (Frontend):** At the bottom of the lesson page, render a prominent `<CaseStudyCard>` that directly links to the associated simulation, with a CTA like "Put Your Knowledge to the Test."
    *   **Action (Frontend):** Inside the `SimulationWorkspace.tsx`, add a "Need a Refresher?" component that links back to the foundational lessons associated with that case's competencies.

#### **Gap 3: Case Studies Feel Disconnected from the Curriculum**
Currently, case studies are just another content type. They should feel like the "final exam" for a set of lessons.

**Solution: Frame Case Studies as Capstone Events**

1.  **Learning Path Integration:** Position case studies as the final step in the "Curated Learning Paths." A path is not complete until the case study is passed.
2.  **Prerequisites:** Your `prisma.schema` has a `prerequisites` field on the `Case` model. Use it. Before a user can start a case, show them the 3-5 lessons they should complete first. This creates a natural learning flow.
3.  **Debrief Tie-in:** The `<RecommendedReading>` component in the debrief is key. Make it more prominent. If a user scores low on a competency, the debrief page should present the relevant lessons as a clear "remedial" action plan.
